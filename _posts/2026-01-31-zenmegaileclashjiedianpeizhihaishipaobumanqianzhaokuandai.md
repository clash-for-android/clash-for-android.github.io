<p>有一个很现实的问题大家可能都遇到过：同一个<strong>机场节点</strong>，在电脑上飞快，到了手机上就慢如蜗牛；或者反过来，<strong>Shadowrocket订阅</strong>在iPhone上跑得飞起，把订阅链接转到电脑Clash上就各种报错。这其实往往不是节点的问题，而是不同客户端对规则集的解析逻辑不一样。</p>

<p>比如我之前有个习惯，为了省电，手机上的CFA我很少开“预解析”和“Keep-Alive”。结果就是每次切APP，网络都要卡顿两三秒才能连上。后来我发现，Clash的核心机制是基于规则分流的，如果你的本地DNS设置跟节点配置里的DNS冲突，或者你的分流规则太旧（比如还在用两年前的GeoIP数据库），那真的是神仙难救。</p>

<p>再比如<strong>便宜的机场</strong>通常为了节约成本，不支持UDP转发。如果你在玩游戏或者用语音通话，Clash默认配置里如果没有把UDP流量屏蔽或者特殊处理，你的体验就是“听得到声音，画面不动”，或者“一直正在连接”。这时候，你如果不去改本地的配置策略，换多少个节点都没用。</p>

<h3>常见问题与真实解决方式</h3>

<p>在各种群里混久了，我发现大家遇到的问题惊人的相似。这里我不讲大道理，就列几个我遇到过并解决的高频问题，全是我自己在键盘上敲出来的血泪经验。</p>

<h4>1. 为什么节点全是绿的，但就是上不去网？</h4>
<p>这是最经典的“假通”现象。通常是因为系统时间不同步，或者端口冲突。</p>
<p><strong>尝试解决：</strong></p>
<p>首先检查你的系统时间，Clash的协议对时间极其敏感，差几十秒都可能导致握手失败。其次，看看是不是端口被占用了。</p>
<code>
# Windows下查看端口占用（以7890为例）
netstat -aon | findstr "7890"

# 如果发现被其他程序占了，去Clash设置里把混合端口改成7891或其他的
</code>

<h4>2. 配置文件导入报错 "YAML Error"</h4>
<p>这个真的能把人气死，特别是自己手搓或者合并订阅的时候。YAML格式对缩进要求极其严格，多一个空格少一个空格都不行。</p>
<p><strong>尝试解决：</strong></p>
<p>不要用记事本打开编辑！一定用VS Code或者Sublime Text。如果你是，尽量用在线的订阅转换工具，把<strong>机场节点</strong>链接转换成标准的Clash订阅链接，不要试图自己去拼凑。</p>

<h4>3. 开了TUN模式后，国内软件反而上不去了</h4>
<p>很多人为了打游戏开了TUN模式（虚拟网卡），结果发现微信发不出图，或者B站加载不出来。这是因为配置里的分流规则没写好，把内网流量也拦截了。</p>
<code>
# 在配置文件中检查是否开启了自动路由系统流量
tun:
  enable: true
  stack: system # 或者 gvisor
  auto-route: true
  auto-detect-interface: true
  dns-hijack:
    - 198.18.0.2:53
</code>
<p>如果不懂怎么改代码，最简单的办法：去下载一个“白名单模式”的配置文件模板，覆盖进去。</p>
